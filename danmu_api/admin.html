📖 快速使用指南

1. 配置 TOKEN

在"环境变量"标签页中设置 TOKEN，用于 API 访问鉴权。默认值为 87654321。

示例： TOKEN=your_secret_token

2. 配置数据源

通过 SOURCE_ORDER 配置数据源查询顺序，支持的源包括：

  • 360 - 360看源
  • vod - VOD采集站
  • tmdb - TMDB源
  • douban - 豆瓣源
  • bilibili - 哔哩哔哩
  • tencent - 腾讯视频
  • iqiyi - 爱奇艺
  • youku - 优酷
  • imgo - 芒果TV
  • renren - 人人视频
  • hanjutv - 韩剧TV
  • bahamut - 巴哈姆特

示例： SOURCE_ORDER=360,vod,bilibili,renren

3. API 使用示例

# 搜索动漫 GET /api/v2/search/anime?keyword=生万物 # 自动匹配 POST /api/v2/match Body: {"fileName": "生万物.S01E01.2160p.WEB-DL"} # 获取弹幕 (通过 URL) GET /api/v2/comment?url=https://v.qq.com/x/cover/xxx.html&format=json # 获取弹幕 (通过 ID) GET /api/v2/comment/10001?format=xml # 查看日志 GET /api/logs

4. 播放器配置

以 senplayer 为例：

  1. 获取部署后的 API 地址，如 http://192.168.1.7:9321/87654321
  2. 在播放器的"设置 - 弹幕设置 - 自定义弹幕API"中填入 API 地址
  3. 播放时点击"弹幕按钮 - 搜索弹幕"，选择你的弹幕 API
  4. 根据标题搜索并选择剧集即可加载弹幕

5. 进阶配置

  • 限流配置：通过 RATE_LIMIT_MAX_REQUESTS 设置 1 分钟内同一 IP 最大请求次数
  • 弹幕过滤：使用 EPISODE_TITLE_FILTER 过滤预告、花絮等非正片内容
  • 缓存配置：设置 SEARCH_CACHE_MINUTES 和 COMMENT_CACHE_MINUTES 控制缓存时间
  • Redis 持久化：配置 UPSTASH_REDIS_REST_URL 和 TOKEN 启用 Redis 存储
